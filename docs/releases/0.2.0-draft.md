# Draft Release Notes: `v0.2.0`

`apntalk/esl-core` remains a pre-`1.0.0`, framework-agnostic FreeSWITCH ESL protocol substrate for PHP. This release prepares the current core for an honest small pre-`1.0.0` tag without expanding it into a runtime or framework integration layer.

## Highlights

- Typed ESL protocol substrate covering framing, parsing, deterministic classification, typed commands, and typed replies
- Stable supported ingress contract via `InboundPipeline`
- Normalized event decoding for `text/event-plain` and `text/event-json`, plus provisional `text/event-xml`
- Selective typed event families for background jobs, channel lifecycle, bridge, hangup, playback, and custom events
- Curated live-backed bridge/playback evidence in both `text/event-plain` and `text/event-json` for the current selective typed event families
- Correlation/session metadata primitives including `ConnectionSessionId`, `ObservationSequence`, `JobCorrelation`, and metadata envelopes
- Replay-safe envelopes that preserve protocol-native facts separately from derived correlation metadata
- Minimal in-memory transport for tests and smoke-path use
- Internal stream/socket smoke validation covering fragmented, coalesced, delayed-body, delayed-completion, and mid-frame-loss scenarios
- Hardened failure taxonomy for malformed, truncated, unsupported, transport, and replay-consistency failures
- Fixture-backed capability verification and clean PHPUnit/PHPStan coverage

## What This Release Does Not Add

- No framework integration layer
- No event-loop ownership or reconnect supervision
- No transport expansion beyond `InMemoryTransport`
- No replay storage, scheduling, or orchestration
- No `1.0.0` API stability promise

## Upgrade/Adoption Notes

- Treat the package as a protocol substrate, not a full ESL runtime.
- Prefer `InboundPipeline` as the upper-layer ingress contract.
- Treat lower-level parser/classifier composition as provisional even when the interfaces remain available for advanced use.
- Treat replay and reconstruction-oriented contracts as pre-`1.0.0` provisional surfaces.
- Unknown events continue to degrade safely to `RawEvent`.
- Treat `tools/smoke/` as non-public operator validation support. It documents evidence generation and environment setup; it is not a supported package API.

## Release readiness

This repository can now be described as:

**v0.2 release-ready core hardening completed, pending maintainer release decision**

Remaining pre-`1.0.0` caveats are intentional and documented:

- XML event decoding is still provisional pending broader evidence
- replay-related surfaces remain provisional
- internal smoke transports and tooling remain non-public
- no runtime/framework/orchestration ownership is implied
